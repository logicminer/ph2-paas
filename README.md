# PH2 PaaS

Self-hosted WordPress + Next.js PaaS on baremetal, behind Cloudflare Tunnel. Works behind CGNAT — no public IP, no open inbound ports.

## What this is

A reproducible setup that turns any Ubuntu 24.04 box into a multi-tenant hosting platform:

- **WordPress sites** → OpenLiteSpeed (OLS) with LSCache, shared MariaDB
- **Next.js / Node apps** → Dokku (git-push-to-deploy)
- **All traffic** → Cloudflare Tunnel (catch-all ingress, outbound-only)
- **Management** → Web panel (Next.js + SQLite) + Portainer, both behind Cloudflare Access (email OTP)

```
Internet → Cloudflare edge (TLS) → cloudflared tunnel (outbound)
  → Dokku Nginx :80 (hostname routing)
      ├─ WP domains → OLS :8088 (shared, LSCache) → MariaDB (shared, per-site DBs)
      ├─ Node apps  → Dokku containers (git push)
      ├─ Panel      → Next.js container (SQLite)
      └─ Portainer  → Docker management UI
```

## Quick start (new machine)

```bash
# 1. Clone this repo
git clone <repo-url> /opt/ph2/paas
cd /opt/ph2/paas

# 2. Configure
cp .env.example .env
nano .env  # fill in Cloudflare creds + domains

# 3. Bootstrap (installs everything, ~10 min)
sudo bash bootstrap.sh
```

After bootstrap completes, visit your panel URL. See [docs/BOOTSTRAP.md](docs/BOOTSTRAP.md) for details.

## Prerequisites

1. **Ubuntu 24.04** (or similar Debian-based) baremetal or VPS
2. **Cloudflare account** with:
   - Your domain(s) added as zones (active, nameservers pointing to CF)
   - An API token with: `Zone → DNS:Edit`, `Account → Cloudflare Tunnel:Edit`, `Account → Access:Apps:Edit`
3. **No public IP required** — works behind CGNAT (the whole point)

## Key design decisions (and why)

| Decision | Why |
|---|---|
| **OpenLiteSpeed for WP (not Nginx+PHP-FPM)** | LSCache is server-level full-page caching — significantly faster than plugin-based caching. Non-negotiable for WP performance. |
| **Shared MariaDB (not per-site containers)** | One DB instance, per-site logical databases. RAM-flat per new client (~0 extra MB vs ~300MB per-container). |
| **Dokku for Node apps** | Git-push-to-deploy, MIT-licensed, no feature gates. Node apps are stateless — fits Dokku perfectly. |
| **Cloudflare catch-all tunnel** | Set-and-forget: new sites need only a DNS CNAME + Dokku/OLS config. Never touch cloudflared per-client. |
| **zram + swap tier** | zram (compressed RAM) as primary swap, disk swap as overflow only. Effectively expands usable RAM ~40% on compressible workloads (PHP/Node). |
| **Panel = Next.js + SQLite** | Lightweight, matches the deployed stack, no external DB needed. Cloudflare Access gates it. |

```

├── .env.example                # All required inputs documented
```

## How the scripts interact

All scripts read shared config from `/etc/ph2/env` (generated by bootstrap). No hardcoded values — the same scripts run identically on every machine.

```
/etc/ph2/env (mode 600, root-only)
  ├── CLOUDFLARE_ACCOUNT_ID, CLOUDFLARE_TOKEN
  ├── DOKKU_GLOBAL_DOMAIN, PANEL_DOMAIN, PORTAINER_DOMAIN
  ├── MARIADB_ROOT_PASSWORD, OLS_ADMIN_PASSWORD
  ├── TUNNEL_ID
  ├── SCRIPTS_DIR, BACKUP_DIR
  └── Container names, versions, memory sizes
```

## Capacity

On a 16GB RAM / 6-core box (zram-boosted to ~22GB effective):
- **~10-14 mixed apps** comfortable (WP + Node)
- **~15-20 WP-only** sites (shared OLS + MariaDB is very efficient)
- RAM is the binding constraint (not CPU or bandwidth)
- 32GB RAM upgrade is the first capacity lever (cheap, 2 DIMM slots)

## Security

- Panel + Portainer behind **Cloudflare Access** (email OTP, 24h session)
- OLS WebAdmin on localhost only (not exposed)
- SSH on port 22 (for Dokku git pushes; LAN-reachable behind CGNAT)
- All secrets in `/etc/ph2/env` (mode 600, root-only) or Docker env vars
- Scripts run via sudoers (scoped to specific scripts, passwordless for automation)
